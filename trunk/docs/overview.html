This document is the API specification for the Titanium Backup Message Viewer application.

This is a Swing GUI application to view the XML files created from the Android App Titanium Backup.

Those message XML files are located under the folder TitaniumBackup on the mobile phone and have following name pattern: com.keramidas.virtual.XML_MESSAGES-YYYYMMDD-HHMMSS.xml.gz

Unfortunately, the XML file does not contain the contact's name. Therefore you can optionally choose a contacts database to look up contact's name from phone number.

Contacts database is stored as {@code contacts2.db} in a archive file with name pattern {@code com.android.providers.contacts-YYYYMMDD-HHMMSS.tar.gz} in the {@code TitaniumBackup} folder on the mobile phone.

MMS Message currently only support message parts with following content type:

  • text/plain
  • text/x-vCard
  • image/jpeg
  • application/smil

SMIL implementation is rudimentary because it does not honor layout description, it only cares for correct order of message parts.

The GUI displays both sides of a messaging (SMS/MMS) conversation on the right screen side in chronological order. Each message is displayed like a conversation bubble in a cartoon.

On the left side a list box of message threads is displayed. In this way it groups messages from the same person and phone number together.
